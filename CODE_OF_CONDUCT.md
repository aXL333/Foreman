# Code of Conduct

Foreman is a safety tool. Project discussion should be practical, respectful, and focused on making agent work more visible and accountable.

## Expected Behavior

- Be direct, technical, and respectful.
- Assume good faith, but ask for evidence when claims affect safety or trust.
- Keep reports actionable: include versions, logs, screenshots, or reproduction steps where possible.
- Redact tokens, credentials, private project names, and sensitive command output.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Posting secrets, private data, or exploit details where a private security report is appropriate.
- Turning detection discussions into runnable offensive tutorials.
- Repeatedly derailing issues or pull requests after maintainers ask to refocus.

## Reporting

For conduct issues, email `jinkaflops@gmail.com`.

For security vulnerabilities, follow [SECURITY.md](SECURITY.md) and report privately through GitHub Security Advisories or email.

Maintainers may edit, hide, or remove comments; close discussions; or block users when needed to keep the project safe and usable.
