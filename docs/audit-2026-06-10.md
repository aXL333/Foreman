Multi-agent audit of functionality, architecture, and security. Method: six specialized
auditors (MCP security, host security, architecture, concurrency/reliability, detection
correctness, product/UX) swept the whole tree read-only; medium+ findings then went to
adversarial verifiers instructed to refute them against the code. 44 agents, ~2.2M tokens.
A rate limit cut the verification pass short: **11 findings are adversarially verified**
(all confirmed real, several severities adjusted down for the local-only threat model);
the rest are high-confidence finder output, unverified — treat them as strong leads.

Threat model baseline (unchanged): Foreman is local-only and not a sandbox. Adversaries
are a rogue/compromised same-user agent process, a malicious third-party MCP server, and
a careless agent. "A same-user process can do X anyway" findings were excluded by design.

## Executive summary

The macro structure is genuinely good: strict downward dependency direction (Core depends
on nothing), pure decision logic consistently extracted and tested (MutePolicy, AuditPolicy,
StartupRegistration, IdleCleanupPolicy), the kill path is PID-reuse-pinned, MCP transport
auth is sound (localhost bind, constant-time bearer token, origin checks, ACL'd token file),
and the recent rename was handled with a proper migration.

The weaknesses cluster in five themes, not dozens of unrelated bugs:

1. **MCP tools trust caller-supplied identity.** Transport auth proves "some authorized
   client", but every tool then trusts `harnessId`/announced client name for routing and
   scoping. Connected harness A can read B's command lines, ack B's Medium/Low alerts,
   reset B's behavior metrics, and answer B's Ask-Harness/cleanup requests. (Verified.)
2. **Alert state has three owners and no lifecycle.** Tray, dashboard, and ForemanState
   each define "active alert" differently; alerts only accumulate (no auto-resolve, no
   bulk ack, unbounded `_alertById`), so the tray goes permanently amber and counts
   disagree. (Partially verified.)
3. **Detection regexes are evadeable at the margins.** Several rules break on realistic
   variations (trailing redirect, pipe chains, caret/backtick obfuscation, separated
   flags); the Codex env-snapshot downgrade is reachable by name-spoofing. (Verified for
   cred-003, net-001, win-001, downgrade gate, PID-keyed hang state.)
4. **Subsystems die silently.** WMI watcher death, MCP port-bind failure, sidecar pipe
   loss, and swallowed async faults all leave Foreman looking healthy while blind.
5. **Secrets are redacted at one egress, leaked at others.** Masking exists for clipboard
   audit prompts only; raw command lines flow to events.log.jsonl at rest, MCP tool
   output, CSV export, and broadcast notifications.

## Strong unverified findings (verification cut by rate limit)

**Reliability / concurrency**
- WMI watchers have no death detection or restart — a single COM fault silently ends process monitoring (`WmiProcessWatcher.cs:50-59`). HIGH.
- `_ = _mcpHost.StartAsync(...)` is fire-and-forget — port-in-use leaves the app running with no MCP and no error surfaced (`App.xaml.cs:144`). HIGH.
- Settings persistence is non-atomic; a corrupt settings.json silently resets ALL config (incl. security posture: mutes, emergency rules) and overwrites the evidence (`SettingsStore.cs:13-31`).
- Second instance crashes on exit: `OnExit` calls `ReleaseMutex()` on a mutex it never acquired (`App.xaml.cs:26-31,169`).
- BehaviorTracker escalation evaluate/assign race can downgrade `CurrentLevel` under alert bursts (`BehaviorTracker.cs:50-64`).
- No global exception backstop; `SendForAuditClick` is an unguarded `async void` (`AlertDetailWindow.xaml.cs:219`).

**Product / UX**
- "Send test alert…" triggers a REAL Emergency escalation: session-long EMERGENCY banner, alarm window — a first-run user's worst possible onboarding (`TrayController.cs:296-315`). HIGH.
- Tray and dashboard disagree on "active alerts"; dashboard feed omits PermissionViolation/NonzeroExit events its own counters include; no bulk-acknowledge → permanent amber.
- CSV export is formula-injectable via agent-controlled command lines (`LogWindow.xaml.cs:196-219`) — prefix-escape `= + - @`.
- Muting a hang alert mutes orphan/exit alerts too: they share `source == "Foreman.Monitor"` scope (`MutePolicy.cs:80-92`).
- Emergency alarm window can open behind the foreground window (skips `WindowActivation.Surface`).
- Log timestamps drop the date although the persisted log spans days; "Clear" only clears the view; persist toggle silently needs a restart.

**Security (host)**
- Raw command lines (incl. secrets) persisted verbatim to `events.log.jsonl`; redaction exists only for clipboard prompts (`EventLogStore.cs:37`).
- MutePolicy guardrail is bypassable by direct settings.json edit (mutes are trusted at read time) — re-validate against `MutePolicy` when loading.
- Opt-in MCP tool scanner is an SSRF primitive: attacker-writable agent config chooses the URL; no private-IP denylist, size/time caps (`McpToolScanMonitor.cs:127`).
- Elevated sidecar nonce passes on the command line (briefly readable pre-handshake); HKCU Run repair adopts a present-but-foreign target as "enabled".

**Detection**
- del-001/del-005 (`rm -rf`) defeated by separated flags (`rm -r -f`) and quoted paths; del-006 PS recursive-delete regex malformed; cred-007/8/12 miss `Copy-Item` forms; FalsePositiveFilter suppresses ALL rules for test-runner-adjacent process names.
- McpInventoryMonitor per-source first-run baseline: an attacker can time a malicious server install to land in the silent baseline.
- HarnessClassifier 'cline' bucket matches any node.exe with the bare substring; escalation level never decays (high-watermark presented as current state).

1. ✅ Cap `_alertById` (mirror the event-queue cap) and add bulk-acknowledge ("Acknowledge all" on the dashboard's ACTIVE ALERTS card).
2. ✅ CsvEscape: prefix `'` on cells starting `= + - @` (or tab).
3. ✅ Fix the unowned `ReleaseMutex` crash (ownership from `createdNew`).
4. ✅ `EscalationAlarmWindow`: call `WindowActivation.Surface`.
5. ✅ "Send test alert" now uses rule `test-001`; BehaviorTracker skips `test-*` rules, so a drill can no longer trigger a real session-long EMERGENCY.
6. ✅ Pattern fixes: dropped `\s*$` from cred-003 (trailing redirect/pipe no longer bypasses); net-001 tolerates pipe-chains (`curl … | tee … | bash`); win-001 covers `pwsh` + shorter payloads (≥16 chars) + quoted payloads. net-003 was already chain-tolerant on inspection. All with regression tests.
7. ✅ MCP bind failure now publishes a High `MonitoringNoticeEvent` (tray red + guidance) instead of failing silently.
8. ✅ Mute scope: hang/orphan/exit mutes are category-scoped (muting hangs no longer silences orphans/exits from the shared `Foreman.Monitor` source).
9. ✅ `/health`: session count removed from the unauthenticated response.
10. ✅ Ask-Harness ownership: anonymous replies (no `harnessId`/`processId`) are rejected — closes the cheap path of verified finding #1; full session-bound identity remains the paradigm fix.

## Paradigm recommendations (in adoption order)

1. **Session-bound harness identity.** At MCP connect, bind an identity to the SSE session
   (operator confirms which harness a session is, or derive from the connecting process);
   tools default to caller-scope, and cross-harness visibility/ack/reset becomes an
   operator-granted "observer" capability. Structurally eliminates the verified findings
   1–4 instead of patching each tool. Tradeoff: a session-registry + small UI; the
   `SseSessionManager` registration hook already exists.
2. **Alert lifecycle + single AlertStateService.** Alerts become stateful
   (open → auto-resolved → acknowledged → expired): hangs auto-resolve when I/O resumes,
   orphans when the process dies, exits expire after N hours. One service owns
   `IsActive`/counts; tray, dashboard, and `ForemanStatus` all consume it. Kills the
   three-owners divergence and the permanent-amber tray.
3. **Watchdog for the watchman.** Each subsystem (WMI watchers, IoPoller, Kestrel, sidecar,
   scan timer) heartbeats into a health model; tray shows "monitoring degraded" amber when
   any goes quiet, with restart attempts for WMI/Kestrel. Converts the silent-death class
   into visible, recoverable states. (Pairs with a `SafeFireAndForget` helper + global
   `DispatcherUnhandledException`/`UnobservedTaskException` handlers.)
4. **SecretRedactor at the event boundary.** Redact once when events/records are
   constructed (Core), so JSONL persistence, MCP tools, CSV export, broadcasts, and any
   future egress inherit masking. Expand patterns (JWT, GitHub/Slack/OpenAI shapes,
   URL-embedded creds).
5. **Detection robustness package.** (a) Normalization pre-pass before matching: strip
   cmd carets/PS backticks, collapse whitespace, decode obvious `-EncodedCommand`;
   (b) a red-team fixtures corpus (per rule: must-match evasion variants + must-not-match
   benign strings) run in CI; (c) standardize all per-process state on the existing
   `(pid, startTime)` Key; (d) anything that *relaxes* detection (Codex downgrade) must
   require a corroborating non-spoofable signal (verified parent chain / exe path).
   The fixtures corpus is the highest-leverage single investment in the product's core value.
6. **AppServices facade + constructible EventBus + atomic SettingsService.** Replace the
   ~44 hand-wired nullable delegates / 14 static window hooks with 3–4 small injected
   interfaces (IProcessQuery, IBehaviorControl, IAskHarness, ISettingsService); give
   EventBus a public ctor (tests stop sharing the singleton); persist settings via
   temp-file + `File.Replace` with corrupt-file quarantine + visible notice. Deliberately
   *not* a full DI container — the app is too small to justify the ceremony.

## Idle Harness self-cleanup (shipped today) — audit interaction

The feature follows the auditor-recommended "escalation ladder" shape (ask → grace →
visible notice; reaping stays manual). Two audit items affect it directly:
- Verified finding #1 (anonymous `ReplyToAskHarnessRequest`) lets a sibling forge the
  "cleaned up" reply — quick-win 10 closes this.
- Suggested future primitive: per-session **last-tool-call timestamp** in
  `SseSessionManager` for a true "connected but MCP-idle" signal alongside I/O silence.
